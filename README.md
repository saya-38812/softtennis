# ソフトテニス サーブ診断アプリ

AIを活用したソフトテニスのサーブフォーム分析・コーチングアプリケーションです。動画をアップロードするだけで、MediaPipeによる骨格検出とOpenAI APIによる詳細なアドバイスを提供します。

## 概要

このアプリケーションは、ソフトテニスプレイヤーのサーブフォームを動画から自動解析し、以下の機能を提供します：

- **骨格検出**: MediaPipeを使用した高精度なポーズ検出
- **フォーム分析**: 成功フォームとの比較による詳細な診断
- **弱点特定**: 3つの重要指標による集中的な分析（MVP版）
- **視覚的比較**: 成功フォームとユーザーフォームの比較画像を自動生成
- **セッション解析**: 5本のサーブ動画から外れ値除去＋平均による安定した診断
- **改善度フィードバック**: 前回セッションとの比較による「どれだけ良くなったか」のコメント
- **練習メニュー提案**: 診断結果に基づいた個別化された練習メニュー（最優先1つ）
- **AIコーチング**: OpenAI GPT-4o-miniによる簡潔なアドバイス（身体感覚の指示＋単一練習）

## 主な機能

### 1. 動画解析（1本ごとの処理）
- MP4形式のサーブ動画をアップロード
- MediaPipeによる骨格検出
- 成功フォーム（`success.mp4`）との比較分析
- インパクトフレーム（腕が一番上の瞬間）の自動検出

### 2. 診断項目（MVP版：3項目に集中）
以下の3つの重要指標を分析します：

- **打点の高さ** (`impact_height`)
  - インパクト時の手首の高さを測定
  - 低すぎる場合は改善が必要

- **肘の角度** (`elbow_angle`)
  - インパクト時の肘の角度を測定
  - 曲がりすぎている場合は改善が必要

- **体軸のブレ** (`body_sway`)
  - 体の軸の左右ブレを測定
  - ブレが大きい場合は改善が必要

システムは3つの指標を評価し、**最も改善が必要な1つの指標**を自動的に選定します。

### 3. 視覚的比較機能
- 成功フォームとユーザーフォームの比較画像を自動生成
- インパクト時のフレームを抽出
- 改善ポイントに応じた視覚的なガイドラインを描画
  - 打点の高さ：横ライン表示
  - 肘の角度：ターゲットマーク表示
  - 体軸のブレ：縦ライン表示（理想軸と現在軸）

### 4. AIコーチング
- 最も悪い指標に応じた「身体感覚コーチング」の自動生成
- **ai_text**: 1行の動作イメージ（比喩表現）
- **practice**: 1行の具体的な単一練習（回数指示付き）

### 5. セッション解析と改善度フィードバック
- 1セッションあたり**5本のサーブ動画**をアップロード
- 5本分のスコアから外れ値をIQRで1回だけ除去し、平均値で最終診断を計算
- 最終診断に対して「前回セッションとの改善度」を自動計算
  - 打点の高さ：前回より何％高くなったか
  - 肘の角度：前回より何度改善したか
  - 体軸のブレ：どれだけブレが小さくなったか
- 改善度に応じて、日本語のフィードバック文（`improvement_message`）を返却

## 技術スタック

### バックエンド (Render)
- **Python 3.x**
- **FastAPI**: RESTful APIサーバー
- **MediaPipe**: ポーズ検出・骨格分析
- **OpenCV**: 動画処理
- **NumPy**: 数値計算
- **OpenAI API**: GPT-4o-miniによるコーチング生成
- **Uvicorn**: ASGIサーバー

### フロントエンド (Vercel)
- **React 19**: UIフレームワーク
- **Axios**: HTTPクライアント（API通信用）
- **CSS3**: モダンなUIデザイン

## セットアップ

### 前提条件
- Python 3.8以上
- Node.js 16以上
- OpenAI APIキー

### バックエンドのセットアップ

1. リポジトリをクローン
```bash
git clone <repository-url>
cd soft-tennis
```

2. バックエンドディレクトリに移動
```bash
cd backend
```

3. 仮想環境を作成・有効化
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS/Linux
python3 -m venv venv
source venv/bin/activate
```

4. 依存パッケージをインストール
```bash
pip install -r requirements.txt
```

5. 環境変数を設定
`backend`ディレクトリに`.env`ファイルを作成し、以下を記述：
```env
OPENAI_API_KEY=your_openai_api_key_here
```

6. MediaPipeモデルファイルの確認
`backend/ai/models/pose_landmarker_full.task`が存在することを確認してください。

7. 成功フォーム動画の確認
`backend/ai/success.mp4`が存在することを確認してください。

### フロントエンドのセットアップ

1. フロントエンドディレクトリに移動
```bash
cd frontend
```

2. 依存パッケージをインストール
```bash
npm install
```

## 実行方法

### バックエンドサーバーの起動

```bash
cd backend
# 仮想環境を有効化
venv\Scripts\activate  # Windows
# source venv/bin/activate  # macOS/Linux

# サーバーを起動
uvicorn main:app --reload --port 8000
```

サーバーは `http://127.0.0.1:8000` で起動します。

### フロントエンドの起動

別のターミナルで：

```bash
cd frontend
npm start
```

ブラウザで `http://localhost:3000` が自動的に開きます。

## APIエンドポイント

### POST `/analyze`
サーブ動画をアップロードして解析を実行します。  
同じセッションID（`X-Session-ID`ヘッダー）で最大5本までスコアを蓄積し、5本揃った時点で最終診断と改善度を返します。

**リクエスト:**
- Content-Type: `multipart/form-data`
- Body: `file` (動画ファイル)

**レスポンス（途中経過：5本未満）:**
```json
{
  "status": "collecting",
  "count": 3
}
```

**レスポンス（5本揃った最終結果）:**
```json
{
  "status": "complete",
  "diagnosis": {
    "weakness": {
      "impact_height": "low",
      "elbow_angle": "ok",
      "body_sway": "unstable"
    },
    "scores": {
      "impact_height": 0.20,
      "elbow_angle": 25.5,
      "body_sway": 0.05
    }
  },
  "ai_text": "ボールを空に押し上げるイメージで腕を伸ばしていきましょう",
  "practice": "トスを高く上げて、空に向かって腕を伸ばす動きを10回繰り返しましょう",
  "focus_label": "打点の高さ",
  "ideal_image": "/outputs/ideal_1234567890.png",
  "user_image": "/outputs/user_1234567890.png",
  "mean_scores": {
    "impact_height": -0.20,
    "elbow_angle": -25.5,
    "body_sway": 0.05
  },
  "improvement": {
    "impact_height": 0.10,
    "elbow_angle": 5.0,
    "body_sway": -0.02
  },
  "improvement_message": "前回より打点が10%高くなっています"
}
```

### POST `/menu-detail`
練習メニューの詳細な練習方法を取得します。

**リクエスト:**
```json
{
  "menu_name": "体重移動練習（強化）",
  "diagnosis": { ... }
}
```

**レスポンス:**
```json
{
  "status": "ok",
  "detail": "具体的な練習方法の説明..."
}
```

## 使用方法

1. **動画の準備**
   - サーブの動画をMP4形式で準備
   - プレイヤー全体が映っている動画が推奨

2. **動画のアップロード（5本）**
   - フロントエンドの「動画ファイルを選択」ボタンから1本ずつ動画を選択
   - 画面上部に「撮影 X / 5」と本数が表示されます
   - 5本選び終わると「診断を開始」ボタンが有効になります

3. **解析の実行**
   - 「診断を開始」ボタンをクリックすると、5本の動画が順番にバックエンドへ送信されます
   - バックエンド側で5本分のスコアを蓄積し、5本揃った時点で平均＋外れ値除去による最終診断を計算します

4. **結果の確認**
   - 成功フォームとユーザーフォームの比較画像が表示されます
   - 最優先の改善ポイント（`focus_label`）が1つ表示されます
   - AIコーチからの身体感覚アドバイス（`ai_text`）と、1つだけの具体的な練習（`practice`）が表示されます
   - 改善度コメント（`improvement_message`）で、前回セッションからどれだけ良くなったかが表示されます
   - 別API（`/menu-detail`）を使うことで、練習メニュー名に対するより詳細な練習説明（最大6行・箇条書き）を取得できます

## 注意事項

- 動画ファイルは解析完了後に自動的に削除されます
- 生成された比較画像は最新5枚のみ保持され、古い画像は自動的に削除されます
- OpenAI APIの使用にはAPIキーが必要です（有料）
- 解析には数秒から数分かかる場合があります
- 動画の品質が低い場合、骨格検出が失敗する可能性があります
- 現在のバージョンは右利きプレイヤーを想定しています
- MVP版のため、診断項目は3項目に絞られています
 - セッション単位で評価するため、**毎回5本まとめて撮影すること**を前提としています

## トラブルシューティング

### 骨格検出が失敗する場合
- 動画の解像度を確認してください
- プレイヤーが動画内に完全に映っているか確認してください
- 照明条件を改善してください

### APIエラーが発生する場合
- `.env`ファイルに`OPENAI_API_KEY`が正しく設定されているか確認してください
- APIキーの有効性を確認してください
- ネットワーク接続を確認してください

### サーバーが起動しない場合
- 仮想環境が正しく有効化されているか確認してください
- 必要なパッケージがすべてインストールされているか確認してください
- ポート8000が使用可能か確認してください
